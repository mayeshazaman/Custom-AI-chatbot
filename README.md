# 📄 Custom AI Chatbot — Chat with Your PDFs using Gemini

A RAG (Retrieval-Augmented Generation) chatbot that lets you upload PDF documents and ask questions about their content. Powered by **Google Gemini**, **LangChain**, and **FAISS** — with a **validation layer** for confidence-scored, grounded answers.

---

## ✨ Features

- 📂 **Multi-PDF Support** — Upload and query multiple PDFs at once
- 🔍 **Semantic Search** — FAISS vector store with Gemini embeddings
- 🤖 **Gemini LLM** — Answers generated by `gemini-2.5-flash`
- ✅ **Validation Layer** — Confidence scoring + grounding check on every answer
- 💬 **React Frontend** — Modern UI with Vite + React
- 🚀 **FastAPI Backend** — REST API with two clean endpoints

---

## Architecture

```
PDF Upload
    │
    ▼
Text Extraction (PyPDF2)
    │
    ▼
Text Chunking (RecursiveCharacterTextSplitter)
    │
    ▼
Embedding (Gemini Embeddings) → FAISS Index (saved to disk)
    │
    ▼
User Query (React Frontend)
    │
    ▼
Similarity Search (Top-4 chunks)
    │
    ▼
LLM Generation (Gemini 2.5 Flash)
    │
    ▼
Validation Layer
    ├── Out-of-scope detection
    ├── Confidence scoring (word overlap)
    └── Fallback if confidence < 30%
    │
    ▼
{ answer, confidence, status } → React UI
```

## ⚙️ Installation

### Prerequisites
- Python 3.9+
- Node.js 18+
- A Google Gemini API key — [get one free here](https://makersuite.google.com/app/apikey)

### 1. Clone the repository

```bash
git clone https://github.com/mayeshazaman/Custom-AI-chatbot.git
cd Custom-AI-chatbot
```

### 2. Set up Python environment

```bash
python -m venv venv

# Windows
venv\Scripts\activate

# macOS / Linux
source venv/bin/activate

pip install -r requirements.txt
```

### 3. Set up environment variables

Create a `.env` file in the root:

```
GOOGLE_API_KEY=your_gemini_api_key_here
```

### 4. Install frontend dependencies

```bash
cd frontend
npm install
cd ..
```

## 🚀 Running the App

### Option 1 — Windows (recommended)

Double-click `run.bat` or run it from the terminal. It starts both servers at once:

```bat
@echo off
start cmd /k "cd /d G:\custom-ai-chatbot && venv\Scripts\activate && uvicorn app:app --reload --port 8000"
start cmd /k "cd /d G:\custom-ai-chatbot\frontend && npm run dev"
```

### Option 2 — Manual (two terminals)

**Terminal 1 — Backend:**
```bash
uvicorn app:app --reload --port 8000
```

**Terminal 2 — Frontend:**
```bash
cd frontend
npm run dev
```

Then open: `http://localhost:5173`

API docs available at: `http://localhost:8000/docs`

## 🔌 API Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| `POST` | `/upload` | Upload one or more PDFs and build the FAISS index |
| `POST` | `/ask`    | Ask a question about the uploaded PDFs |

### POST `/upload`

**Request:** multipart form with one or more PDF files

**Response:**
```json
{
  "message": "Processed 2 file(s) into 14 chunks. Index is ready."
}
```

### POST `/ask`

**Request:**
```json
{
  "question": "What is the refund policy?"
}
```

**Response:**
```json
{
  "answer": "The refund policy allows returns within 30 days of purchase...",
  "confidence": 0.82,
  "status": "ok"
}
```

**Possible status values:**

| Status | Meaning |
|--------|---------|
| `ok` | Answer is grounded and confident |
| `low_confidence` | Answer score below 30% threshold |
| `no_docs` | No relevant chunks found for the query |
| `no_index` | No FAISS index — PDFs not uploaded yet |

## Validation Layer

Every answer goes through a two-step check before being returned:

**1. Out-of-scope detection** — if the LLM signals it couldn't find the answer in the context (e.g. "outside scope of the uploaded documents"), the answer is flagged.

**2. Confidence scoring** — measures what fraction of meaningful words in the answer appear in the retrieved document chunks. Stopwords (`the`, `a`, `is`, etc.) are excluded.

```python
confidence = matched_words / total_answer_words
```

If either check fails (confidence < 0.30), a safe fallback message is returned instead of a potentially hallucinated answer.
