# 📄 Custom AI Chatbot — Chat with Your Documents

A RAG-based document chatbot with a **FastAPI** backend and **React** frontend. Upload PDF, TXT, or DOCX files and ask questions — answers are grounded strictly in your documents using **Groq (LLaMA 3.3)** and **HuggingFace embeddings**.

---

## Features

- Upload PDF, TXT, and DOCX files as your knowledge base
- Automatic text extraction, chunking, and FAISS vector indexing
- Semantic search powered by `all-MiniLM-L6-v2` embeddings (runs locally, no extra API key)
- Answers generated by `llama-3.3-70b-versatile` via Groq
- Multi-session conversation history with context window (last 6 messages)
- Confidence scoring to flag low-quality or out-of-scope answers
- Incremental indexing — new uploads merge into the existing FAISS index
- REST API with `/upload`, `/ask`, `/clear/{session_id}`, and `/health` endpoints
- Request/response logging to `logs/app.log`

## Setup

**1. Clone the repository**
```bash
git clone https://github.com/mayeshazaman/Custom-AI-chatbot.git
cd Custom-AI-chatbot
```

**2. Install dependencies**
```bash
pip install -r requirements.txt
```

**3. Create the logs directory**
```bash
mkdir logs
```

**4. Add your Groq API key**

Create a `.env` file in the project root:
```
GROQ_API_KEY=your_groq_api_key_here
```
Get a free key at [console.groq.com](https://console.groq.com).

**5. Run the backend**
```bash
uvicorn app:app --reload --port 8000
```

**6. Run the React frontend**

From your frontend directory:
```bash
npm install
npm start
```

## API Endpoints

| Method | Endpoint | Description |
|---|---|---|
| `POST` | `/upload` | Upload one or more files to build/update the index |
| `POST` | `/ask` | Ask a question (supports `session_id` for multi-turn chat) |
| `DELETE` | `/clear/{session_id}` | Clear conversation history for a session |
| `GET` | `/health` | Check server status and whether the index is ready |

### Example: Ask a question
```bash
curl -X POST http://localhost:8000/ask \
  -H "Content-Type: application/json" \
  -d '{"question": "What is the refund policy?", "session_id": "user_1"}'
```

Response:
```json
{
  "answer": "The refund policy states...",
  "confidence": 0.87,
  "status": "ok"
}
```

## Notes

- The FAISS index is saved to `faiss_index/` and persists across restarts
- New uploads are **merged** into the existing index, not replaced
- Chunk size is 4,000 characters with 400-character overlap
- Answers with confidence below 0.3 are flagged as `low_confidence`
- File size limit is 10MB per file; supported types are `.pdf`, `.txt`, `.docx`
- HuggingFace embeddings run locally — no external API call needed for retrieval
